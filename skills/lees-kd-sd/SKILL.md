# Skill: SBB Kwalificatiedossier Software Developer

## Doel

Deze skill maakt de inhoud van het SBB kwalificatiedossier Software Developer (crebo 25998, niveau 4, versie 2024) beschikbaar als referentie. Gebruik de inhoud van dit KD altijd als je vragen beantwoordt over:

- Wat studenten moeten kennen en kunnen
- Welke kerntaken en werkprocessen van toepassing zijn
- Welke gedragsindicatoren horen bij een werkproces
- Welke competenties worden aangesproken
- Hoe opdrachten, toetsen of lessen aansluiten op het KD

## Gebruik

Lees het volledige KD via het referentiedocument:

📄 `references/kd-software-developer.md`

Lees dit bestand altijd voordat je een antwoord geeft dat gebaseerd moet zijn op het KD.

## Structuur van het KD

Het KD bestaat uit:

- **Basisdeel (B)** — geldt voor alle studenten Software Developer
  - **B1-K1: Realiseert software** — 5 werkprocessen (W1 t/m W5)
  - **B1-K2: Voert ICT-projecten uit** — 3 werkprocessen (W1 t/m W3)
- **Profieldeel (P)** — één profiel: Software Developer, niveau 4
- **Generieke onderdelen** — taal (3F), rekenen (3F), L&B, Engels (B1/A2)

Elk werkproces bevat:
- Een **omschrijving** van de activiteit
- Een **resultaat** (wat er aan het einde moet zijn)
- **Gedragsindicatoren** (observeerbaar gedrag)
- **Competenties** (met code en naam)

## Aanwijzingen voor gebruik

- Verwijs altijd naar het juiste werkproces met de officiële code (bijv. B1-K1-W3)
- Gebruik de exacte formulering van gedragsindicatoren bij toetsing of beoordeling
- Bij het koppelen van een opdracht aan het KD: benoem expliciet welke werkprocessen en gedragsindicatoren worden aangesproken
- Bij onduidelijkheid over welk werkproces van toepassing is: raadpleeg het referentiedocument en redeneer vanuit het beroepsresultaat
